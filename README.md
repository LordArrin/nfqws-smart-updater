# nfqws-smart-updater
Продвинутый скрипт обновления IP-листов для [NFQWS](https://github.com/Anonym-tsk/nfqws-keenetic) (OpenWrt). Умная агрегация подсетей (CIDR merging), строгая валидация (sipcalc), кэширование, умное использование ресурсов устройства и **интеграция с Home Assistant по MQTT**. Поддерживает cron и объединение локального файла с удалёнными источниками.

---

# 📖 Инструкция по обновлению списков NFQWS (OpenWrt).

(!) Keenetic/Entware не тестировались.

Этот скрипт автоматически скачивает списки IP-адресов сайтов и CDN (сервера которых стремительно деградируют), очищает их, удаляет дубликаты, объединяет подсети и обновляет конфигурацию **NFQWS**.

### 🛠 Шаг 1. Установка необходимых компонентов

Перед запуском скрипта нужно установить системные утилиты. Скрипт использует продвинутые инструменты для сортировки, математики, параллельной обработки и отправки метрик, которых нет в стандартной прошивке OpenWrt по умолчанию.

Зайдите в терминал устройства (SSH) и выполните команды:

OpenWrt 24.10 и более старые:
```
opkg update
opkg install curl coreutils-sort coreutils-sleep coreutils-nl coreutils-md5sum coreutils-split sipcalc flock gawk bash iprange coreutils-timeout findutils-xargs mosquitto-client-ssl
```

OpenWrt 25.12 и новее, включая Snapshot:
```
apk --update-cache add curl coreutils-sort coreutils-sleep coreutils-nl coreutils-md5sum coreutils-split sipcalc flock gawk bash iprange coreutils-timeout findutils-xargs mosquitto-client-ssl
```

---

### 📥 Шаг 2. Установка скрипта

Предположим, мы будем хранить всё в папке `/etc/nfqws/scripts`.

1.  **Создайте файл скрипта:**
    Сохраните файл скрипта `update_list.sh` из Releases и переместите на нужное вам устройство любимым способом - к примеру, WinSCP для Windows.

2.  **Откройте Терминал и подключитесь к вашему устройству по SSH:**
    ```
    ssh root@IP.адрес.устройства
    ```

3.  **Сделайте скрипт исполняемым:**
    ```
    chmod +x /etc/nfqws/scripts/update_list.sh
    ```

---

### 📝 Шаг 3. Подготовка списка источников

Создайте текстовый файл, в котором будут ссылки на списки IP-адресов или доменов.
Назовем его `urls.txt`.

```
nano /etc/nfqws/lists/urls.txt
```

**Пример содержимого `urls.txt`:**
```
# Список от V3nilla (пример)
https://raw.githubusercontent.com/v3nilla/ipsets-for-bypass-in-russia/refs/heads/main/ipset-all.txt

# Ваши личные списки (если есть URL)
https://example.com/my-list.txt
```
*Можно добавлять комментарии через символ `#`.*

### 🚀 Шаг 4. Ручной запуск (Проверка)

Попробуем запустить скрипт вручную, чтобы убедиться, что всё работает.

Вам нужно знать, куда NFQWS ожидает получить итоговый файл. Обычно это `/etc/nfqws/ipset.list`.

**Команда запуска:**
```
/etc/nfqws/scripts/update_list.sh -f /etc/nfqws/lists/urls.txt -o /etc/nfqws/ipset.list -w /tmp/nfqws_updater/urls/
```

**Основные параметры:**
*   `-f ...` — путь к файлу со ссылками на источники.
*   `-o ...` — куда сохранить готовый результат.
*   `-w ...` — рабочая папка для скрипта. По умолчанию это `/tmp/nfqws_updater/`.
В этой папке скрипт хранит **кэш** скачанных списков (папка `/cache`), создает временные файлы во время обработки и сортировки (папка `/temp`) и хранит файл статистики (`.stats`), чтобы помнить статистику прошлого запуска. Для каждого файла со ссылками нужна отдельная папка.

`Если вам плевать на статистику и защиту от сбоев, можно использовать одну папку, но добавить флаг -S (Skip Safety Check), чтобы скрипт не паниковал из-за скачков количества строк. Но тогда отчёты "Last updated" и "Diff" будут показывать бессмыслицу.`

**Дополнительные параметры:**
*   `-S` — **Отключить защиту (Safety Check)**.
    *   *Когда использовать:* Если вы кардинально поменяли источники и новый список стал намного (10%<) меньше старого (скрипт по умолчанию заблокирует такое изменение, на тот случай, если это сбой).
*   `-R` — **Без перезагрузки сервиса**.
    *   *Когда использовать:* Если вы хотите просто обновить файл, но не хотите, чтобы NFQWS перезапускался прямо сейчас (например, чтобы не рвать активные соединения).
*   `-e <файл>` — **Добавить локальный файл**.
    *   *Пример:* У вас есть файл `default_ips.txt` на роутере, который вы правите вручную. Можно "подмешать" его к скачанным спискам: `-e /etc/nfqws/lists/default_ips.txt`. Файл может быть только один.
*   `-t <секунды>` — **Пауза между загрузками** (по умолчанию 3 сек).
    *   *Когда использовать:* Чтобы избежать блокировки по IP (Rate Limit) со стороны серверов, с которых вы скачиваете списки.
*   `-M <файл>` — **Включить отправку статистики в MQTT (Home Assistant)**.
    *   *Пример:* `-M /etc/nfqws/mqtt.conf`. Подробнее см. Шаг 6.
*   `-D` — **Запуск без применения изменений (Dry Run)**.
    *   *Когда использовать:* На случай недостатка уверенности в себе. Никакие файлы не будут перезаписаны, сервис не перезапустится.

### ⏰ Шаг 5. Автоматизация (Cron)

Чтобы списки обновлялись сами (например, каждую ночь), добавим задачу в планировщик.

1.  Откройте редактор Cron:
    ```
    crontab -e
    ```

2.  Добавьте строку (например, обновление в 4:00 утра каждый день):
    ```
    0 4 * * * /etc/nfqws/scripts/update_list.sh -f /etc/nfqws/lists/urls.txt -o /etc/nfqws/ipset.list
    ```

    **Вариант без перезагрузки сервиса (с флагом -R):**
    Если вы хотите обновлять файл часто, но перезагружать сервис редко, можно сделать так:
    ```
    # Обновляем файл каждые 5 часов БЕЗ перезагрузки
    0 */5 * * * /etc/nfqws/scripts/update_list.sh -R -f /etc/nfqws/lists/urls.txt -o /etc/nfqws/ipset.list
    
    # Перезагружаем сервис раз в сутки (командой самого роутера)
    5 4 * * * service nfqws-keenetic restart
    ```

### 🏠 Шаг 6. Интеграция с Home Assistant (MQTT)

Скрипт умеет отправлять метрики (статус обновления, количество IPv4/IPv6, размер файла) напрямую в Home Assistant через MQTT Auto-Discovery.

1. Создайте конфигурационный файл для MQTT:
    ```
    nano /etc/nfqws/mqtt.conf
    ```
2. Впишите туда данные от вашего MQTT брокера:
    ```
    MQTT_HOST="192.168.1.100"
    MQTT_PORT="1883"
    MQTT_USER="ваш_пользователь"
    MQTT_PASS="ваш_пароль"
    ```
3. Добавьте флаг `-M /etc/nfqws/mqtt.conf` к вашей команде запуска (вручную или в cron). После успешного запуска в Home Assistant автоматически появится новое устройство `NFQWS Updater` с сенсорами!

### ❓ Частые вопросы и решение проблем

**1. Скрипт пишет "SAFETY TRIGGER: New list size is too small" и не обновляет файл.**
*   **Причина:** Новый список стал меньше старого более чем на 10% (защита от того, что сайт-источник отдал пустой или битый файл). *Примечание: при самом первом запуске скрипта эта защита автоматически смягчается.*
*   **Решение:** Если вы уверены, что список должен уменьшиться, запустите один раз вручную с флагом `-S`:
    `/etc/nfqws/scripts/update_list.sh -S ...`

**2. Скрипт пишет "Failed to restart service".**
*   **Причина:** Имя сервиса или команда в ОС в скрипте не совпадает с ожиданиями.
*   **Решение:** Скрипт использует стандартную для OpenWrt команду `service` и название предпочтительного сервиса `nfqws-keenetic`. Если у вас другая ОС или сервис, команда может называться иначе. Скрипт обновит файл, но перезагрузить сервис придётся вручную.

**3. Хватит ли оперативной памяти или производительности процессора?**
*   Скрипт умный: он сам проверит доступную память и использует не более 50% (но не более 4Гб) памяти и оставит одно ядро свободным (если их больше одного), чтобы не "повесить" устройство. Для быстрой склейки файлов используется оптимизированный пайплайн (`xargs`).

**4. Что будет, если скрипт запустится дважды одновременно?**
*   Ничего страшного. В скрипте реализована строгая защита от параллельного запуска (`flock`). Второй инстанс просто завершит работу, увидев, что обновление уже идёт.

**5. Где логи?**
*   Скрипт пишет краткий отчет в `/tmp/firewall_update.log`. Можно проверить командой: `cat /tmp/firewall_update.log`.

**6. Не износится ли флеш-память от частой перезаписи?**
*   Скрипт хранит все временные и кешированные файлы в оперативной памяти - по умолчанию в папке `/tmp/nfqws_updater/`. Можно указать любую папку в качестве рабочей с помощью параметра `-w`.
